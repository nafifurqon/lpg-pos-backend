## Objective / Summary

Integrate Swagger (OpenAPI) into the NestJS backend so all existing endpoints are
fully documented â€” request bodies, response schemas, auth requirements, and error responses.
Swagger UI is accessible at `/docs`.

## Scope

### In Scope

- Install `@nestjs/swagger`
- Bootstrap Swagger UI in `main.ts` at `/docs`
- Annotate all controllers with `@ApiTags`, `@ApiOperation`, `@ApiResponse`, `@ApiBearerAuth`
- Add `@ApiProperty` / `@ApiPropertyOptional` (manual, no CLI plugin) to all DTOs
- Create `AuthResponseDto` + `UserInAuthDto` response DTO classes
- Document error envelope `{ messages: string[], error: string }`
- Document success envelope `{ message: string, result: T }`

### Out of Scope

- OpenAPI spec export to file
- Auto-generated SDK clients
- NestJS Swagger CLI plugin

## Acceptance Criteria

- [x] Swagger UI accessible at `http://localhost:6000/docs`
- [x] All 8 endpoints visible across 3 tags: Auth, Shops, Dashboard
- [x] Request body schemas shown for all POST endpoints
- [x] Auth endpoints reference `AuthResponseDto` schema
- [x] Protected endpoints show lock icon (Bearer)
- [x] `npx tsc --noEmit` passes with no errors
